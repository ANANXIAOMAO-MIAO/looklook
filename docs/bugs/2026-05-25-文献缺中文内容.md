# 文献没有中文标题和总结

**日期**：2026-05-25  
**环境**：线上 looklook.onrender.com  
**状态**：代码已修复，待部署到 Render 后验证

---

## 用户看到什么

搜索结果里，有的文献只有英文标题，没有中文标题、一句话总结、深度点评、亮点和不足。  
有的文献是正常的。不是整次搜索都坏了。

典型表现：列表里英文标题下面又出现一行同样的英文，总结区域是空的。

---

## 我们查到了什么

Render 日志里有一条：

```text
[JSON解析失败] 原始返回前200字符: {
```

后面能看到模型其实已经开始写中文了（`title_cn`、`summary_cn`），但 `summary_cn` 写到一半就断了，JSON 没有写完。

同一时间还有：

```text
正在生成中文摘要与深度点评...（第 3/3 批）
```

说明问题出在 **生成中文内容的第 3 批**，前面几批多半是成功的。

和用户是否登录、是否下载 Excel **没有关系**。

---

## 原因（大白话）

系统会一次让 AI 给最多 5 篇文献写：中文标题、摘要、80～150 字的点评、亮点、不足。内容很长。

AI 回答有长度上限。当时代码 **没有设置 `max_tokens`**，最后一批写太长时，回答被 **截断**，只剩半截 JSON（例如 summary 停在「揭示了」）。

程序解析不了这半截 JSON，就当成「这一批全部失败」，用英文标题顶上，总结等就空了。

所以会出现：**同一页里有的文献有中文，有的没有**。

---

## 本次改了什么（2026-05-25）

| 改动 | 说明 |
|------|------|
| `max_tokens` | P3 中文生成用 8192；其它 JSON 阶段（意图解析、相关度、综述）用 4096 |
| 每批篇数 | 从 5 篇改为 **3 篇**（`BATCH_SIZE_ENRICH = 3`） |
| 失败重试 | 某批有文献缺中文 → **整批再试一次** → 还不行就 **一篇一篇** 再试 |
| 日志 | `[JSON解析失败]` 会带上 `stage=P3`、`batch=几/几`、`len=返回长度` |
| 失败提示 | 不再用英文标题冒充中文标题，改为显示「（中文标题生成失败，请见上方英文标题）」 |

相关代码：`main.py` 里的 `call_deepseek`、`parse_json_response`、`enrich_papers_batch`、`run_pipeline`。

---

## 部署后怎么验证

1. 把最新代码部署到 Render。  
2. 用容易筛出很多相关文献的兴趣描述搜一次（最好能出现多批 P3，例如 3/3 批）。  
3. 看列表里是否每篇都有中文标题和总结。  
4. 看 Render 日志：  
   -  ideally 不再出现 `[JSON解析失败]`；  
   - 若出现，看是否有 `[P3] ... 整批重试` / `单篇重试` 且最终文献仍有中文。

---

## 若部署后仍有个别篇目失败

日志里搜 `[P3] ... 仍有 N 篇未能生成中文材料`，把该次搜索的兴趣描述和 task 时间段记下来，可再考虑：

- 进一步把每批改成 2 篇；  
- 或缩短 `recommendation_text` 的字数要求。
